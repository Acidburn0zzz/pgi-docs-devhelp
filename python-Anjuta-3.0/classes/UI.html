add_action_group_entries(action_group_name, action_group_label, entries, translation_domain, can_customize, user_data)
Parameters:
  • action_group_name (str) – Untranslated name of the action group.
  • action_group_label (str) – Translated label of the action group.
  • entries ([Gtk.ActionEntry]) – An array of action entries.
  • translation_domain (str) – The translation domain used to translated the entries. It is usually the GETTEXT_PACKAGE macro in a project.
  • can_customize (bool) – If true the actions are customizable by user.
  • user_data (object) – User data to pass to action objects. This is the data that will come as user_data in “activate” signal of the actions.
Returns:

A Gtk.ActionGroup object holding all the action objects.
Return type:

Gtk.ActionGroup

Gtk.Action objects are created from the Gtk.ActionEntry structures and added to the UI Manager. “activate” signal of Gtk.Action is connected for all the action objects using the callback in the entry structure and the user_data passed here.

This group of actions are registered with the name action_group_name in Anjuta.UI. A Gtk.Action object from this action group can be later retrieved by Anjuta.UI.get_action () using action_group_name and action name. action_group_label is used as the display name for the action group in UI manager dialog where action shortcuts are configured.
