Abstract:No
Structure:IBus.ProxyClass

An IBus.Proxy is the base of all proxy objects, which communicate the corresponding IBus.Services on the other end of IBusConnection. For example, IBus clients (such as editors, web browsers) invoke the proxy object, IBus.InputContext to communicate with the InputContext service of the ibus-daemon.

Almost all services have corresponding proxies, except very simple services.

destroy()

Dispose the proxy object. If the dbus connection is alive and the own variable above is True (which is the default), org.freedesktop.IBus.Service.Destroy method will be called. Note that “destroy” signal might be emitted when IBus.Proxy.destroy is called or the underlying dbus connection for the proxy is terminated. In the callback of the destroy signal, you might have to call something like ‘g_object_unref(the_proxy);’.
